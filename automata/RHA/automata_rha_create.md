## 2. JSON Schema 分析

通过阅读 `CreatData.py`、`HybridAutomata.py`、`ODE_System.py`、`DE.py` 和已有的 JSON 样例（`ball.json`、`tanks.json`、`oscillator.json`、`duffing.json` 等），确定 schema 结构如下：

```json
{
  "automaton": {
    "var": "x1, x2",                     // 逗号分隔的状态变量名
    "input": "u1",                        // 可选，逗号分隔的输入变量名
    "mode": [
      {
        "id": 1,                          // 整数模式 ID
        "eq": "x1[1] = expr, x2[1] = expr"  // ODE 方程, x[k] 表示 k 阶导数
      }
    ],
    "edge": [
      {
        "direction": "1 -> 2",            // 源模式 -> 目标模式
        "condition": "x1 <= 0",           // Python 布尔表达式（守卫条件）
        "reset": {                        // 可选，重置映射
          "x1": [0],                      //   数值常量
          "x2": ["-0.9 * x2[0]"]          //   或字符串表达式
        }
      }
    ]
  },
  "init_state": [                         // 15 条初始状态（前 6 条用于测试，后 9 条用于训练）
    {"mode": 1, "x1": [1.0], "x2": [0.5]},
    ...
  ],
  "config": {
    "dt": 0.01,              // 离散时间步长
    "total_time": 10.0,      // 总仿真时间
    "order": 1,              // ODE 阶数
    "other_items": "x_[?]",  // 额外基函数（用于特征工程）
    "kernel": "rbf",         // SVM 核函数类型（可选）
    "need_reset": true,      // 是否需要重置（可选）
    "self_loop": true         // 是否允许自环（可选）
  }
}
```

**关键发现**：
- `condition` 通过 `eval('lambda ' + var + ':' + condition)` 解析为 Python lambda
- 支持 `and`/`or` 等 Python 逻辑运算符
- `other_items` 中 `x[?]` 展开为各阶状态，`x_[?]` 展开为其他变量
- 输入（`input`）在 `init_state` 中以时间 `t` 的表达式给出（如 `"0.1 * sin(0.3 * t)"`）

## 3. 逐系统转换方法

### 3.1 BouncingBall

**MATLAB 定义**：单模式线性混合系统，自环转换模拟弹跳。

| 项目 | MATLAB | JSON |
|---|---|---|
| 动力学 | `A=[0 1;0 0], c=[0;-9.81]` | `x1[1] = x2[0], x2[1] = -9.81` |
| 守卫 | `conHyperplane([1,0],0)` + `x2<=0` | `x1 <= 0` |
| 重置 | `reset.A=[0,0;0,-0.9], reset.c=[0;0]` | `"x1":[0], "x2":["-0.9 * x2[0]"]` |
| 初始区域 | `interval([0.1;-1],[1;1])` | 15 点在 `[0.1,1]×[-1,1]` 内 |
| 仿真参数 | `tFinal=2` | `dt=0.01, total_time=2.0` |

**特殊处理**：设置 `self_loop: true` 和 `need_reset: true`。

### 3.2 VanDerPol

**MATLAB 定义**：纯连续非线性系统（Van der Pol 振荡器），无混合切换。

| 项目 | MATLAB | JSON |
|---|---|---|
| 动力学 | `f = @(x,u) [x(2); (1-x(1)^2)*x(2)-x(1)]` | `x1[1] = x2[0], x2[1] = (1 - x1[0]**2) * x2[0] - x1[0]` |
| 边 | 无 | `"edge": []` |
| 初始区域 | `interval([-3;-3],[3;3])` | 15 点在 `[-3,3]×[-3,3]` 内 |
| 仿真参数 | `tFinal=6` | `dt=0.01, total_time=6.0` |

**`other_items`**：`"x[?] ** 2 * x_[?]"` — 捕获 `x1^2 * x2` 非线性项。

### 3.3 JetEngine

**MATLAB 定义**：Moore-Greitzer 喷气发动机模型，纯连续非线性。

| 项目 | MATLAB | JSON |
|---|---|---|
| 动力学 | `f = @(x,u) [-x(2)-1.5*x(1)^2-0.5*x(1)^3; 3*x(1)-x(2)]` | `x1[1] = -x2[0] - 1.5 * x1[0]**2 - 0.5 * x1[0]**3, x2[1] = 3 * x1[0] - x2[0]` |
| 边 | 无 | `"edge": []` |
| 初始区域 | `interval([-3;-3],[3;3])` | 15 点在 `[-3,3]×[-3,3]` 内 |

**`other_items`**：`"x[?] ** 2; x[?] ** 3"` — 捕获二次和三次项。

### 3.4 Oscillator（切换振荡器）

**MATLAB 定义**：4 模式线性混合系统，循环切换 1→2→3→4→1。

所有模式共享系统矩阵 `A = [-2, 0; 0, -1]`，但常数项 `c` 不同：

| 模式 | 区域（不变集） | 常数项 c | JSON 方程 |
|---|---|---|---|
| 1 | x1≥0, 0.714x1+x2≥0 | [1.4, -0.7] | `-2*x1[0]+1.4, -x2[0]-0.7` |
| 2 | x1≥0, 0.714x1+x2≤0 | [-1.4, 0.7] | `-2*x1[0]-1.4, -x2[0]+0.7` |
| 3 | x1≤0, 0.714x1+x2≤0 | [-1.4, 0.7] | `-2*x1[0]-1.4, -x2[0]+0.7` |
| 4 | x1≤0, 0.714x1+x2≥0 | [1.4, -0.7] | `-2*x1[0]+1.4, -x2[0]-0.7` |

**守卫条件推导**（从 CORA polytope/hyperplane 到 Python 布尔表达式）：

```
MATLAB: guard = conHyperplane([-0.714286 -1], 0)
含义:   轨迹穿过超平面 -0.714286*x1 - x2 = 0

从模式 1 (0.714x1+x2 >= 0) 向下穿越:
JSON:   "0.714286 * x1 + x2 < 0"

从模式 2 (x1 >= 0) 向左穿越:
JSON:   "x1 < 0"

从模式 3 (0.714x1+x2 <= 0) 向上穿越:
JSON:   "0.714286 * x1 + x2 > 0"

从模式 4 (x1 <= 0) 向右穿越:
JSON:   "x1 > 0"
```

### 3.5 PiecewisePoly1

**MATLAB 定义**：3 模式非线性混合系统，沿坐标轴划分状态空间。

| 模式 | 区域 | 动力学 | JSON 方程 |
|---|---|---|---|
| A (id=1) | x2 ≥ 0 | dx1=-7, dx2=-x1 | `-7, -x1[0]` |
| B (id=2) | x2<0, x1≥0 | dx1=0.5x1²+0.5x2, dx2=-9x1+3 | `0.5*x1[0]**2+0.5*x2[0], -9*x1[0]+3` |
| C (id=3) | x2<0, x1<0 | dx1=5, dx2=-0.1x1-10 | `5, -0.1*x1[0]-10` |

**守卫条件**：使用 `and` 组合条件：
- `1 -> 2`: `"x2 <= 0 and x1 >= 0"` （穿过 y=0 进入第四象限）
- `1 -> 3`: `"x2 <= 0 and x1 < 0"` （穿过 y=0 进入第三象限）
- `2 -> 1`: `"x2 >= 0"` （穿过 y=0 回到上半平面）
- `2 -> 3`: `"x1 <= 0"` （穿过 x=0 进入第三象限）
- `3 -> 1`: `"x2 >= 0"`
- `3 -> 2`: `"x1 >= 0"`

### 3.6 PiecewisePoly2

**MATLAB 定义**：4 维 2 模式非线性混合系统，沿 x3=0 超平面切换。

| 模式 | 区域 | 动力学差异 |
|---|---|---|
| 1 | x3 ≤ 0 | dx4 = -0.1*(x3+5)² |
| 2 | x3 ≥ 0 | dx4 = -0.1*(x1-x3)² |

两模式共享 dx1=-x2, dx2=x1-5, dx3=x4。参数：s=1, r1=5, r2=5, p=0.1。

### 3.7 FillingTanks（三水箱填充系统）

**MATLAB 定义**：3 模式线性混合系统。原始 3D 状态通过线性变换 `z = Tx + b` 投影到 2D。

**坐标变换计算**：

```
T = [-1/sqrt(2),  1/sqrt(2), 0        ;
     -1/sqrt(6), -1/sqrt(6), 2/sqrt(6)]

b = -T * [1/3; 1/3; 1/3] = [0; 0]
```

**动力学常数向量**（投影后为恒定流）：

| 模式 | 原始流 | 投影后 (x1_dot, x2_dot) |
|---|---|---|
| Fill Tank 1 | [0.8, -0.5, -0.3] | (-1.3/√2, -0.9/√6) ≈ (-0.919239, -0.367423) |
| Fill Tank 2 | [-0.2, 0.5, -0.3] | (0.7/√2, -0.9/√6) ≈ (0.494975, -0.367423) |
| Fill Tank 3 | [-0.2, -0.5, 0.7] | (-0.3/√2, 2.1/√6) ≈ (-0.212132, 0.857321) |

**守卫条件计算**：不变集为三角形 polytope，顶点：

```
v1 = T*[1;0;0] = (-0.7071, -0.4082)   // tank 1 满
v2 = T*[0;1;0] = (0.7071, -0.4082)     // tank 2 满
v3 = T*[0;0;1] = (0, 0.8165)           // tank 3 满
```

三条边的半空间约束（外法线形式 n·z ≤ d）：

| 边（某水箱为空） | 法线 | 阈值 d | JSON 守卫条件 |
|---|---|---|---|
| Tank 1 空（右边 v2-v3） | (√3/2, 0.5) | 1/√6 ≈ 0.408248 | `0.866025*x1 + 0.5*x2 >= 0.408248` |
| Tank 2 空（左边 v1-v3） | (-√3/2, 0.5) | 1/√6 ≈ 0.408248 | `-0.866025*x1 + 0.5*x2 >= 0.408248` |
| Tank 3 空（底边 v1-v2） | (0, -1) | 1/√6 ≈ 0.408248 | `x2 <= -0.408248` |

### 3.8 TwoRoomHeater

**MATLAB 定义**：4 模式线性混合系统 + 环境温度输入。

所有模式共享 `A = [-0.2, 0.1; 0.1, -0.2]`, `B = [0.1; 0.1]`，常数项 c 根据加热器开关状态不同：

| 模式 | 加热器 1 | 加热器 2 | c |
|---|---|---|---|
| 1 | ON | ON | [6, 5] |
| 2 | OFF | ON | [0, 5] |
| 3 | ON | OFF | [6, 0] |
| 4 | OFF | OFF | [0, 0] |

**恒温器逻辑**（T_on=20, T_off=21）：

```
模式 1 → 2: x1 >= 21  (Room 1 过热，关闭加热器 1)
模式 1 → 3: x2 >= 21  (Room 2 过热，关闭加热器 2)
模式 2 → 1: x1 <= 20  (Room 1 降温，开启加热器 1)
模式 2 → 4: x2 >= 21
模式 3 → 4: x1 >= 21
模式 3 → 1: x2 <= 20
模式 4 → 3: x1 <= 20
模式 4 → 2: x2 <= 20
```

**输入处理**：环境温度 u1 以正弦函数表达式提供，范围 [5, 15]。

### 3.9 Aircraft

**MATLAB 定义**：3 状态 2 输入非线性连续系统（着陆机动）。

**动力学方程**（m=60000kg, g=9.81m/s²）：

```
D(x,u) = (2.7 + 3.08*(1.15 + 4.2*u2)^2) * x1^2     // 阻力
L(x,u) = 68.6*(1.25 + 4.2*u2) * x1^2                 // 升力

dx1/dt = (1/m) * (u1*cos(u2) - D(x,u) - m*g*sin(x2))
dx2/dt = (1/(m*x1)) * (u1*sin(u2) + L(x,u) - m*g*cos(x2))
dx3/dt = x1 * sin(x2)
```

JSON 方程直接编码上述表达式，使用 Python `cos()`/`sin()`/`**` 语法。

**输入处理**：MATLAB 中输入为限速随机序列，JSON 中简化为正弦函数 `"u1": "1000 * sin(0.5 * t)"`，保持在有效范围 `[-5000, 5000]` 和 `[-0.25, 0.25]` 内。

## 4. 验证方法

验证分为 6 个维度，共覆盖 9 个基准系统的全部转换。

### 4.1 Schema 验证

检查每个 JSON 文件的结构完整性：
- 顶层键 `automaton`, `init_state`, `config` 存在
- `automaton.mode[]` 中每个条目有 `id`(int) 和 `eq`(str)
- `automaton.edge[]` 中每个条目有 `direction` 和 `condition`
- `init_state` 恰好 15 条，每条包含 `mode` 和所有状态变量
- `config` 中所有键属于合法集合 `{dt, total_time, order, window_size, ...}`

### 4.2 动力学正确性

对每个模式，在已知状态点计算 JSON ODE 的数值导数，与 MATLAB 方程的解析导数对比：

**方法**：
1. 将系统重置到测试状态点
2. 用极小时间步 `dt=1e-7` 做一步 RK4 积分
3. 计算 `(state_after - state_before) / dt` 作为数值导数
4. 与手算的 MATLAB 解析导数对比，计算相对误差
5. 容限：相对误差 < 0.1%

**测试点选取**（举例）：

| 基准 | 模式 | 测试点 | 预期导数 |
|---|---|---|---|
| BouncingBall | 1 | x1=0.5, x2=1.0 | [1.0, -9.81] |
| VanDerPol | 1 | x1=1.0, x2=2.0 | [2.0, -1.0] |
| JetEngine | 1 | x1=2.0, x2=1.0 | [-11.0, 5.0] |
| Oscillator | 1 | x1=0.5, x2=0.3 | [0.4, -1.0] |
| FillingTanks | 1 | x1=0, x2=0 | [-1.3/√2, -0.9/√6] |
| TwoRoomHeater | 1 | x1=10, x2=15, u=10 | [6.5, 4.0] |
| Aircraft | 1 | x1=75, x2=-0.05, x3=100, u1=0, u2=0 | 解析计算 |

### 4.3 守卫条件正确性

在边界点和内部点评估守卫表达式，验证布尔返回值正确：

**方法**：对每条边，选取：
- 已知应触发转换的点（expected=True）
- 已知不应触发转换的点（expected=False）

共测试 49 个点，覆盖全部有守卫条件的系统。

**举例**（Oscillator）：

```
Edge 1->2 ("0.714286*x1 + x2 < 0"):
  (1.0, -1.0) -> True   // 0.714*1+(-1) = -0.286 < 0
  (1.0,  0.0) -> False   // 0.714*1+0 = 0.714 > 0
```

### 4.4 初始状态有效性

验证每个初始状态的 `mode` 赋值与该模式不变集（invariant region）一致：

**不变集定义**（从 MATLAB polytope 提取）：

| 系统 | 模式 | 不变集 |

### 4.5 数据生成质量

通过 `CreatData.py` 实际生成数据并检查：
- 文件数量（预期 15 个 `.npz`）
- NaN/Inf 检测
- 数值发散检查（|value| < 1e10）
- 轨迹长度一致性（≈ total_time / dt）

对比 JSON `config` 参数与 MATLAB 仿真参数（`dt`, `tFinal`, 系统阶数）。

### 5.5 告警分析

**告警 1：BouncingBall 未检测到模式切换**

- **原因**：自环转换 `1→1` 不改变 mode ID，`np.diff(mode)` 始终为 0
- **验证**：`change_points` 数组记录了共 57 次弹跳重置事件；高度在弹跳后确认反弹
- **结论**：行为正确。自环重置通过 `change_points` 而非 mode 数组记录

**告警 2：PiecewisePoly1 无模式切换**

- **原因**：模式 A 的解析解 `x2(t) = x2_0 - x1_0*t + 3.5*t²`，最小值 `x2_min = x2_0 - x1_0²/14`
  - 初始区域 `[1,2]×[1,2]` 中，最坏情况 `x2_min = 1.0 - 4/14 = 0.714 > 0`
  - 因此 x2 永远不过零，守卫 `x2 <= 0` 永不触发
- **数据验证**：所有轨迹中 x2 最小值 = 0.794，与解析结果一致
- **MATLAB 对照**：MATLAB 使用相同初始区域 `R0=interval([1;1],[2;2])`，行为完全一致
- **结论**：转换正确，这是基准设计的固有特性（测试算法从单区域数据中识别边界的能力）

## 7. 使用方法

在 `Dainarx-pq/` 目录下运行：

```python
from main import main
```
